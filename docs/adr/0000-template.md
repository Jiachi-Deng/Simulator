# ADR 0000: 决策标题

- Status: Proposed
- Date: YYYY-MM-DD
- Owners: GitHub handle or team
- Issue: https://github.com/Jiachi-Deng/Simulator/issues/NUMBER

## Context

描述用户问题、当前约束、已有证据，以及不作决定会造成什么影响。明确区分已确认事实和假设。

## Decision

用可测试、无歧义的语言描述选择。列出适用范围和明确不在范围内的内容。

## Alternatives Considered

列出真正评估过的替代方案、优点、缺点和拒绝原因。

## Consequences

记录正面影响、维护成本、兼容性变化、迁移要求和不可逆部分。

## Security, Privacy, and Licensing

说明权限、credential、network、filesystem、用户数据、第三方代码和分发权利是否改变。无变化时也要明确写出。

## Rollout and Rollback

说明 feature flag、分阶段上线、数据迁移、失败恢复和回滚条件。

## Verification

列出 tests、CI、artifact inspection、人工验收或运行指标。只有可以复现的证据才能作为验收依据。
