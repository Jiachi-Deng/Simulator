# Simulator 隐私政策

最后更新：2026-07-13

Simulator 是一个本地优先的开源桌面应用。本政策说明本仓库构建的 Simulator 如何处理数据。第三方自行修改或重新分发的构建不在本政策承诺范围内。

## 本地数据

Simulator 会在本机保存运行应用所需的数据，例如 workspace 配置、会话记录、用户创建的文件、日志、模块状态和连接配置。具体目录由操作系统和应用配置决定。卸载应用不一定会自动删除这些数据；删除前请先导出需要保留的内容。

Simulator 可以调用本机进程、读取用户明确选择的目录，并运行用户安装或启用的 Module、Skill、MCP server 和其他工具。这些能力可能访问其授权范围内的数据。

## 网络与第三方服务

Simulator 不承诺完全离线运行。下列操作会把完成请求所需的数据发送到用户选择或配置的服务：

- 调用 LLM provider、API、MCP server 或其他 Source；
- 使用 OAuth 登录或第三方集成；
- 打开外部链接、共享内容或使用需要联网的工具；
- 下载依赖、更新、Module 或其他 Artifact。

发送哪些数据、保存多久以及第三方如何使用数据，取决于对应服务的条款和隐私政策。用户应在连接服务前检查其 endpoint、权限和数据政策。Simulator 不会因为某个 provider 可配置，就代表该 provider 受到 Simulator 背书。

## Crash Reporting 与遥测

本仓库的公开 unsigned 构建默认不包含 crash-reporting ingest URL，因此不会自动把 crash report 上传到维护者控制的服务。构建校验会把 `SENTRY_ELECTRON_INGEST_URL` 等嵌入变量从公开产物中清空。

开发者或第三方分发者可以构建启用了 crash reporting 的自定义版本。此类版本必须在分发前说明数据接收方、采集字段、保留期和用户选择方式；它不属于本仓库当前公开构建的隐私承诺。Simulator 当前没有产品分析遥测承诺，也不会在没有公开政策与用户明确选择的情况下为正式公开构建启用上传。

本机日志仍可能包含文件路径、错误信息或 provider 返回内容。提交 Issue 或安全报告前，请先检查并移除 token、API key、个人信息和不相关的工作内容。

## 凭据

Simulator 会尽可能通过操作系统安全存储或受限的本地配置保存凭据，但并非所有第三方工具都使用同一种存储方式。Module 和外部进程不应获得超出其声明权限的凭据。请勿把凭据写入仓库、Issue、日志附件或公开截图。

## 保留、导出与删除

本仓库维护者不会因为用户运行公开 unsigned 构建而收到一份集中式 Simulator 用户数据库。保留和删除主要发生在本机以及用户主动连接的第三方服务中。

删除本地数据前应关闭 Simulator 及其 Module daemon，并确认需要保留的项目已经导出。第三方服务中的数据必须按照对应服务提供的方式删除。

## 儿童与受监管数据

Simulator 不是专门面向儿童的服务。除非用户已经建立适当的组织政策、访问控制和第三方协议，否则不应使用 Simulator 处理法律要求特殊保护的数据。

## 政策变更与联系

影响公开构建数据行为的变更必须通过本仓库 PR、Review 和 CI。安全问题请遵循 [SECURITY.md](SECURITY.md)；一般问题可以使用 GitHub Issue。正式隐私联系邮箱和产品域名尚未确定，在确定前不会编造或公布临时地址。
